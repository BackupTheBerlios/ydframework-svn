Appendix A: Change log

2004-05-05

  • [new] Added caching to the YDUrl->getContents function.
  • [new] If debugging is specified in the URL using YD_DEBUG, the total size and list of include files is now shown as well.
  • [new] Added an option called $cache to the YDUrl->getContents function which allows you to enable and disable caching.
  • [change] Added the missing include for YDStringUtil in the YDF2_init.php file.
  • [change] The source of the different files are no longer included in the documentation.

2004-05-04

  • [new] Added the YDIpUtil module which contains TCP/IP related functions.
  • [new] Added the YDUrl class which contains an object wrapper for an url.
  • [new] Added the YDStringUtil module which contains string related functions.
  • [new] Added the YDFSImage class which is an object wrapper around an image. This class additionally has the option to create cached thumbnails.
  • [new] Added the YDFSDirectory class which is an object wrapper around a filesystem directory.
  • [new] Added the YDFSFile class which is an object wrapper around a filesystem file.
  • [new] Added the YDFileUtil module which does some common file related functions, such as matching a file name against a pattern.
  • [new] Added the YDArrayUtil module with some common array functions. Currently, it only has the convertToTable function.
  • [change] Moved some of the functions to static functions in the different class modules. We now have the YDObjectUtil and YDPhpUtil classes supporting this.
  • [change] Renamed YDDebug to YDDebugUtil and made the YDIsError function a static method of the YDError object.
  • [change] The examples are now put into their own directory.
  • [change] Made some small tweaks to how the API documentation is generated.
  • [change] The initialization of the Yellow Duck Framework now determines the os name in uppercase to find out the path delimiter.
