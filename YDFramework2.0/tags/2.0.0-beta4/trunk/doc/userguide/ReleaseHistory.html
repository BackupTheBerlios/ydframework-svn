Yellow Duck Framework 2.0

Release History


This is the overview of the different releases.

22 october 2004 - Yellow Duck Framework 2.0 beta 4 (build 582)


  • Added the removeElement function to the form class.
  • Added the label_html and error_html elements to the YDForm::toArray function result.
  • Added ignore patterns and class filters to the YDFSDirectory::getContents function.
  • Improved the speed of both the YDBBCode class and the YDStringUtil::encodeString functions.
  • Added the YDFormElement_DateSelect, YDFormElement_DateTimeSelect and YDFormElement_TimeSelect form elements.
  • Added support for null values in update and insert SQL statements in the YDDatabaseDriver class.
  • Added the YDImport function to include parts of the Yellow Duck Framework instead of relying on the include function from PHP.
  • Added the getBasename and getAbsolutepath functions to the YDFSDirectory class.
  • Added the createDirectory function to the YDFSDirectory class.
  • Added the YDCart class that describes a shopping cart.
  • Added the deleteDirectory function to the YDFSDirectory class.
  • Added license information to the Yellow Duck Framework.
  • Added the download function to the YDFSFile class which forces the browser to download the file.
  • Fixed a huge number of smaller bugs (more info in the changelog)


31 july 2004 - Yellow Duck Framework 2.0 beta 3 (build 483)


  • We are now sponsered by Red Carpet Host
  • The documentation is now completed.
  • Added options to log debugging info and log messages to a file.
  • Much nicer layout for some of the internal index pages and the index page of the examples.
  • Release history instead of the build logs.
  • Updated phpThumb to version 1.4.6.
  • YDFSDirectory::getContents is able to return custom objects now.
  • Removed the build directory as it was not needed anymore.
  • YDDatabaseDriver_mysql is no longer a separate include file, which increases performance (as most people tend to use MySQL? anyway).
  • The API of YDBrowserInfo was updated and it's now part of the YDUtil.php include file.
  • The template class is refactored and is no longer automatically instantiated with each request. Also caching is working fine now. An example for caching was added as well.
  • The Smarty Light libraries are updated to 2.2.0.
  • Added conditional HTTP using etags to the YDFeedCreator module (description)
  • Caching needs to be done with only a cache ID and not a filename (but both should be possible).


18 july 2004 - Yellow Duck Framework 2.0 beta 2 (build 450)


  • Switch from Smarty to Smarty Light for the template engine.
  • PHP 5 compatible, but doesn't yet take advantage of the new features.
  • All features from the underlying template engine are now available in the YDTemplate class.
  • Added constant YD_CRLF which contains the end of line appropriate for the current platform.


15 july 2004 - Yellow Duck Framework 2.0 beta 1 (build 343)


  • First public beta version
